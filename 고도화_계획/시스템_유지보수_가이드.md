# 🏨 사내 리조트 잔여객실 통합 대시보드 시스템 유지보수 가이드 (System Maintenance & Architecture Guide)

본 문서는 사내 임직원 복지용 4대 리조트(리솜, 한화, 소노, 롯데)의 잔여 객실 데이터를 수집하여 통합 대시보드로 가시화하는 시스템의 설계 구조, 핵심 기술, 운영 방법 및 트러블슈팅 가이드를 상세히 기술합니다.

## 1. 시스템 아키텍처 개요 (System Architecture)

본 시스템은 **데이터 수집(Data Crawling) -> 전처리 및 파일 생성(Normalization & Compression) -> 웹 가시화(Presentation)**로 이어지는 단방향 파이프라인으로 구성되어 있습니다.

## 3. 브랜드별 크롤링 기술 메커니즘 (Crawler Mechanics)

리조트 웹사이트의 렌더링 방식 및 보안 수준에 따라 맞춤형 수집 기법이 적용되어 있습니다. 모든 크롤러는 로그인 후 브라우저 내 자바스크립트를 직접 평가(Evaluation)하여 **백엔드 REST API를 가로채거나 직접 비동기 병렬 요청(Fetch)하는 초고속 기법**을 사용합니다. (브라우저 클릭 유도 방식 대비 **10~20배 속도 향상**)

### 3.1. 리솜 리조트 수집기 (`resom_crawler.py`)
* **핵심 기술**: Playwright Headless + HTTP Request Interception + Browser-side parallel `fetch`
* **로그인 & 세션**: ID/PW 입력 및 로그인 완료 후, 예약을 시도하여 회원권 번호(`EH55801000`)를 선택하는 시점에 발생하는 API 요청을 감시합니다.
* **API 가로채기**: `/api/user/reservation/roomReservation/selectCondos` 요청에서 **Authorization 토큰, login-id, user-device** 등의 필수 인증 헤더를 동적으로 추출합니다.
* **병렬 쿼리**: 추출한 인증 토큰을 활용하여 브라우저 내에서 자바스크립트 `Promise.all` 기반의 비동기 fetch 요청을 날려 3개월 치 지점/평형별 예약 캘린더 데이터(`/api/user/reservation/roomReservation/calendarRooms`)를 5초 내외로 수집합니다.

### 3.2. 소노 호텔앤리조트 수집기 (`sono_crawler.py`)
* **핵심 기술**: Playwright Headless + 대표회원번호 동적 조회 + Browser-side `fetch`
* **로그인 & 세션**: 소노 통합 회원 로그인을 수행한 뒤, 잔여 객실 조회 페이지(`/reserve/remaining/pc`)로 진입합니다.
* **회원번호 획득**: 백엔드 REST API인 `/api/hms/user/management/member/reserve/representativeMemberNo`를 브라우저 내에서 `fetch` 호출하여 사용자의 고유 대표 회원번호(`memNo`)를 획득합니다.
* **병렬 쿼리**: 획득한 대표 회원번호와 함께 `/api/hms/user/memberReservation/room/list/remaining` API로 향후 3개월치(약 90일) 날짜에 대한 요청을 15개 청크 단위로 병렬 전송합니다.
* **필터 및 중복 제거 조건**: 예약 상태 코드(`rsvStatusCd`)가 `"A"`(예약원활) 및 `"E"`(마감임박)인 데이터 모두를 필터링하여 수집합니다. 또한, 여러 회원권 옵션 및 패키지로 인해 동일 지점/날짜/객실에 중복 데이터가 들어올 경우, 가장 잔여 객실 수(rsvRmCnt)가 많은 행을 기준으로 중복을 제거하여 수집합니다.

### 3.3. 롯데 리조트 수집기 (`lotte_crawler.py`)
* **핵심 기술**: Playwright Headless + SSO Redirect 처리 + Browser-side `fetch`
* **로그인 & 세션**: 롯데 L.Point 및 리조트 통합 로그인 페이지를 거쳐 세션 연동(SSO)을 처리합니다.
* **세션 활성화**: 세션 확보를 위해 객실 현황 페이지(`/main/ko/reservation/room-list`)로 강제 전환합니다.
* **병렬 쿼리**: 3개월 간의 기간과 보유한 2개의 회원권번호 조합을 바탕으로 `roomList.json` API 호출용 자바스크립트 비동기 요청을 병렬 전송합니다.
* **가용 객실 계산**: API 결과 필터 내에서 `TOTAL_CNT` (총 객실 수)와 `USE_CNT` (사용 중인 객실 수)를 빼서 남은 객실 수(`avail_cnt`)를 자체 계산하고, 마감 플래그(`CLOSE_FLG`)가 `"N"`이면서 잔여가 0개 초과인 경우만 추출합니다.

### 3.4. 한화 리조트 수집기 (`hanhwa_crawler.py`)
* **핵심 기술**: Selenium Headless + Chrome DevTools Protocol (CDP) + Performance Log Interception
* **로그인 & 세션**: 1단계 통합 ID/PW 로그인 수행 후, 2단계 회원권 비밀번호 입력 팝업을 돌파하고 booking 페이지로 세션을 전달합니다.
* **네트워크 인터셉트**: 한화리조트는 자바스크립트 fetch 직접 접근을 방어하므로, Selenium의 Chrome 성능 로그(`performance` log) 감시 모드를 켭니다. 
* **구동 방식**: 브라우저 내에서 16개 리조트 탭과 3개월 조회 드롭다운을 스크립트로 직접 선택 및 검색(`calSch`) 버튼 클릭을 실행합니다. 이로 인해 브라우저가 실행하는 `/doExecute.mvc` 통신을 CDP를 통해 감시하여 JSON 결과 텍스트(`Network.getResponseBody`)를 직접 가로채 파싱합니다.
* **필터 조건**: `RSRV_POSBL_YN`이 `"Y"`인 레코드들을 파싱하여 예약가능수로 환산합니다.

## 4. 데이터 전처리 및 압축 아키텍처 (`convert_to_html.py`)

수집된 이기종 엑셀 파일을 정규화하고 하나의 파일로 병합 및 초경량화하는 빌드 엔진입니다.

### 4.1. 공통 표준 스키마화 (Schema Standardization)
각 크롤러가 작성한 raw 엑셀 파일을 가져와 아래의 **10대 표준 스키마**로 강제 매핑합니다.
* `수집일시`, `브랜드`, `리조트명`, `지역`, `년월`, `일`, `요일`, `객실타입`, `예약가능수`, `요금`

### 4.2. 정제 가공 핵심 로직
1. **과거 날짜 데이터 소거**: 수집 실행일자(현재 시간) 기준 이전 날짜에 해당하는 객실 카드는 루프 내에서 날짜 패턴(정규식) 매칭 후 즉시 필터링하여 대시보드 용량을 세이브합니다.
2. **요일 및 년월 표준화**: 월 표기를 `YYYY.MM`로 고정하고, 한 자리 일자(`04` -> `4`)를 통일시킵니다. 요일 정보가 유실된 경우 타겟 날짜의 타임스탬프를 계산하여 요일을 역산해 채워 넣습니다 (`compute_weekday`).
3. **명칭 및 6대 권역 표준화 (`map_region`)**:
   지점명 텍스트 및 raw 지역 컬럼을 분석하여 다음과 같이 6대 권역으로 맵 매핑을 강제 적용합니다.
   * `강원` (속초, 델피노, 비발디파크, 삼척 등)
   * `경기/서울` (고양, 용인, 산정호수, 더플라자 등)
   * `충청` (대천, 부여, 단양, 천안, 덕산, 안면도, 제천 등)
   * `경상` (김해, 해운대, 청송, 거제, 경주, 남해, 오시리아 등)
   * `전라` (변산, 진도, 여수 등)
   * `제주` (제주, 아트빌라스 등)
4. **리솜 객실 중복 방지**: 리솜 리조트의 경우 평형과 객실타입이 쪼개져 표기되어 중복 매칭되는 문제를 방어하기 위해 두 텍스트를 연동 결합하여 고유 문자열을 만듭니다.
5. **롯데 객실타입 공식 홈페이지 명칭 표준화**: 롯데 리조트의 이해하기 어려운 데이터베이스 약어(예: `h훼미리(T)`, `Deluxe Double(H)`, `LOTTY&LORRY 33`)를 공식 웹사이트 명칭(`클린 패밀리 트윈`, `콘도 디럭스 더블 배리어프리`, `캐릭터 스위트 로티&로리` 등)으로 변환하여 사용자 친화적으로 정제합니다.
   * `H` 또는 `H ` 접두사 -> `클린` (공식 홈페이지 클린형 표기 준수)
   * `C` 또는 `C ` 접두사 -> `콘도` (공식 홈페이지 콘도형 표기 준수)
   * `P` 또는 `P ` 접두사 -> `펫동반`
   * 괄호 표기 변환: `(D)` -> `더블`, `(T)` -> `트윈`, `(F)` -> `패밀리`, `(H)` -> `온돌`, `(A)` -> `타입A`
   * `LOTTY&LORRY` -> `캐릭터 패밀리/스위트 로티&로리`
   * `Deluxe Double(H)` / `DLH` -> `콘도 디럭스 더블 배리어프리` (배리어프리 전용 매핑)

### 4.3. 2차원 배열 기반 대용량 JSON 압축 (61.4% 용량 다이어트)
웹 로딩 시 네트워크 부하와 렌더링 성능 저하를 방지하기 위해 데이터를 초경량화하였습니다.
* **기존 방식 (Object Array)**: 2.93 MB (Key-Value 딕셔너리 포맷)
* **개선 방식 (Compact 2D Array)**: **1.13 MB** (값 리스트 포맷)
* **압축 매커니즘**: JSON 내의 키(Key) 선언을 전부 제거하고, 순수하게 값으로만 이루어진 2차원 배열 구조로 추출합니다. 웹 브라우저 클라이언트 JS 영역에서 매핑 상수(`BRAND = 0`, `RESORT = 1`, `REGION = 2` 등)를 선언하여 인덱스로 고속 액세스하도록 설계했습니다.

## 5. 프론트엔드 기능 및 UI/UX 스펙

최종 생성되는 [resort_availability.html](file:///D:/휴양소/resort_availability.html)은 CSS와 JS가 모두 인라인화된 **100% Standalone (무의존성) 단일 페이지**입니다.

### 5.1. UI/UX 디자인 테마
* **Glassmorphism Theme**: 심해 어두운 블루/네이비 톤(`#090b11`)의 그라데이션 다크 모드를 기본으로 채용하고 반투명 유리 블러(`backdrop-filter: blur(12px)`) 스타일을 적용해 하이테크 & 프리미엄 감각을 표현했습니다.
* **브랜드 고유 컬러 브랜딩**: 
  대시보드 카드와 태그에 각 리조트 브랜드 고유 컬러를 테마 색상으로 할당하여 시각적 인지도를 극대화했습니다.
  * 리솜: 에메랄드 그린 (`#10b981`)
  * 한화: 오렌지 (`#f97316`)
  * 소노: 블루 (`#3b82f6`)
  * 롯데: 크림슨 레드 (`#f43f5e`)

### 5.2. 인터랙티브 기능 및 JavaScript 엔진
1. **다단계 연동 필터 (Cascading Filter)**:
   특정 필터를 선택하면 다른 필터(지역 -> 지점 -> 월 -> 일)들이 현재 걸러진 데이터셋 내에서만 존재하는 유효 옵션으로 자동 실시간 재구축됩니다 (`getFilteredDataExcluding`). 이전 필터 교체로 인한 선택 해제 시 재귀 호출을 통해 목록 모순을 원천 차단합니다.
2. **무한 스크롤링 렌더링 (Intersection Observer)**:
   수만 건의 데이터 카드 엘리먼트를 브라우저에 한 번에 그리면 브라우저 프리징(Lag)이 발생합니다. 이를 극복하고자 화면 스크롤이 하단 센티널(`#sentinel`) 영역에 닿을 때마다 60건씩 쪼개어 카드를 추가 렌더링하는 가상 스크롤 아키텍처를 도입했습니다.
3. **브랜드 자동 연동 카드**:
   사용자가 드롭다운에서 특정 지점을 직접 고를 경우, 상단 대시보드의 브랜드 카드 상태를 감지하여 해당 브랜드를 액티브 처리해 주는 스마트 트리거를 탑재했습니다.

## 6. 개발 유지보수 및 운영 방법 (Maintenance Manual)

### 6.1. 환경 구축 및 필수 패키지 설치
크롤러와 정규화 빌드 엔진을 구동하기 위한 개발 환경 세팅입니다. Python 3.8 이상이 권장됩니다.

```bash
# 1. 필수 패키지 설치
pip install pandas openpyxl python-dotenv playwright selenium webdriver_manager python-dateutil

# 2. Playwright 브라우저 엔진 바이너리 설치 (크롤러 필수 단계)
playwright install chromium
```

### 6.2. 환경 설정 파일 (`.env`) 관리
로컬 환경의 루트 폴더(`D:\휴양소\.env`)에 계정 정보 파일을 위치시킵니다. 보안을 위해 이 파일은 Git에 저장하거나 노출시키면 안 됩니다.

```ini
# 리솜 리조트 계정
RESOM_ID=your_resom_id
RESOM_PW=your_resom_pw

# 소노 호텔앤리조트 계정
DAEMYUNG_ID=your_sono_id
DAEMYUNG_PW=your_sono_pw

# 롯데 리조트 계정
LOTTE_ID=your_lotte_email_id
LOTTE_PW=your_lotte_pw

# 한화 리조트 계정 (2단계 회원권 비밀번호 포함)
HANHWA_ID=your_hanhwa_id
HANHWA_PW=your_hanhwa_pw
HANHWA_MEMBERSHIP_PW=your_hanhwa_membership_password
```

### 6.3. 수집 스케줄러 자동화 (GitHub Actions & Pages)
매번 로컬에서 배치 파일을 실행하지 않고 서버리스로 24시간 자동 수집할 수 있도록 **GitHub Actions** 워크플로우를 연동해 두었습니다.

1. **GitHub Secrets 설정**:
   리포지토리 **Settings -> Secrets and variables -> Actions**로 이동하여 위의 `.env` 키-값 쌍을 그대로 등록합니다.
2. **스케줄러 설정**:
   GitHub Actions 워크플로우에 의해 평일(월~금) 08시, 10시, 12시, 14시, 16시(KST 기준)에 크론탭 스케줄러가 자동 기동되어 코드를 실행합니다.
3. **자동 배포**:
   빌드가 완료되어 갱신된 `resort_availability.html` 파일은 **GitHub Pages** 기능에 의해 지정된 웹 도메인 주소로 즉시 자동 퍼블리싱됩니다. (레포지토리 Settings -> Pages 메뉴에서 빌드 브랜치를 `main`으로 활성화해 두어야 합니다.)

## 7. 장애 대처 및 트러블슈팅 (Troubleshooting)

### 7.1. 특정 리조트의 수집 데이터가 누락되거나 에러가 날 때
* **원인**: 해당 리조트 사이트의 로그인 화면 양식이 바뀌었거나, REST API의 요청 주소(URL) 및 파라미터가 변경된 경우입니다.
* **대처 방법**:
  1. 크롬 브라우저에서 개발자 도구(F12)를 열고 네트워크 탭을 활성화합니다.
  2. 수동으로 로그인 및 객실 조회를 실행하여 요청이 들어가는 API 엔드포인트 URL과 페이로드(Payload), 헤더(Headers) 값을 확인합니다.
  3. 해당 크롤러 파일(예: `lotte_crawler.py` 내의 `collect_all` 주소나 `tasks` 생성부)의 URL 파라미터 규칙 또는 JSON 파싱 키를 갱신합니다.

### 7.2. 드롭다운 필터 선택 시 목록이 나타나지 않거나 렌더링 오류가 날 때
* **원인**: 데이터 전처리 단계(`convert_to_html.py`의 `generate`)에서 내보내는 JSON 컬럼의 배열 인덱스 순서와 자바스크립트 프론트엔드의 매핑 상수가 어긋났을 때 발생합니다.
* **대처 방법**:
  1. `convert_to_html.py`의 `export_cols` 컬럼 리스트 정의 순서와 HTML 템플릿 하단 `<script>` 영역 내부의 상수 매핑 인덱스(`BRAND = 0`, `RESORT = 1`, `REGION = 2` 등) 순서가 1:1로 일치하는지 대조 및 정렬합니다.
  2. 자바스크립트 루프 내부에서 속성 식별자(예: `x.지역`) 형태의 구식 참조가 남아있는지 확인하고, 모두 `x[REGION]` 같은 인덱스 상수 포맷으로 맞춥니다.

### 7.3. 파일 보관 주기 조정이 필요할 때
* **대처 방법**:
  모든 크롤러와 빌더 스크립트 상단에 정의된 `KEEP_DAYS = 7` 상수를 원하는 보관 일수(예: `14` 또는 `30`)로 변경하여 저장합니다. 로컬 빌드 시 이 기간이 경과된 파일들은 스크립트 실행 말미에 안전하게 자동 제거됩니다.
